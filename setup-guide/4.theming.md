# Theming — GTK, Qt, matugen, and greetd

This guide configures the visual theming stack from top to bottom — GTK,
Qt, and the matugen color pipeline — then themes the greetd login screen
and enables it as the final step.

**Prerequisites:** `3.dotfiles.md` complete. Hyprland running with your
real stowed configs. All core packages installed.

---

## Understanding the theming stack

Three separate systems need to be configured independently. GTK themes have
no effect on Qt apps, Qt themes have no effect on GTK apps, and neither
affects Electron apps.

| App type | Themed by | Config location |
| --- | --- | --- |
| GTK3 apps | nwg-look + gtk.css | `~/.config/gtk-3.0/` |
| GTK4 apps | nwg-look + gtk.css (limited) | `~/.config/gtk-4.0/` |
| Qt5 apps | qt5ct + Kvantum | `~/.config/Kvantum/` |
| Qt6 apps | qt6ct + Kvantum | `~/.config/Kvantum/` |
| Waybar | CSS @import | `~/.config/waybar/` |
| Rofi | rasi CSS | `~/.config/rofi/` |
| Swaync | CSS file | `~/.config/swaync/` |
| Kitty | kitty.conf colors | `~/.config/kitty/` |
| Electron | Per-app CSS / settings | App-specific |
| greetd / ReGreet | GTK4 CSS | `/etc/regreet.css` |

matugen generates a color palette from your wallpaper and writes it into
all of these via template files. The Stow-managed config files import the
generated colors rather than containing color values directly — so the
configs stay in git and the generated output never does.

---

## 1. Choose a wallpaper

All colors in this setup derive from your wallpaper via matugen. Choose one
before doing anything else — it becomes the seed for the entire palette.

```bash
mkdir -p ~/pictures/wallpapers

# Copy your wallpaper into place — use any image you like
# jpg, png, and webp all work with matugen
```

Set it with swww:

```bash
swww img ~/pictures/wallpapers/wallpaper.jpg \
  --transition-type wipe \
  --transition-duration 1
```

## 2. GTK theming

### 2.1 Install GTK theme and icons

```bash
sudo pacman -S adw-gtk-theme papirus-icon-theme
```

> **adw-gtk-theme:** renamed from `adw-gtk3`, now in the official `extra`
> repo. Theme files inside kept their original names (`adw-gtk3-dark`) so
> nwg-look and gsettings still reference them by the old name. Makes GTK3
> apps visually consistent with GTK4/libadwaita apps.

### 2.2 Apply GTK settings with nwg-look

Launch nwg-look from a terminal inside Hyprland:

```bash
nwg-look
```

Configure each tab:

| Tab | Setting | Value |
| --- | --- | --- |
| Widget theme | Theme | `adw-gtk3-dark` |
| Icon theme | Icons | `Papirus-Dark` |
| Mouse cursor | Cursor | Your preference (e.g. `Bibata-Modern-Ice`) |
| Font | Font | `Sans 11` |

Click **Apply** when done. nwg-look writes to both dconf and the INI files
so GTK apps pick up the settings without a running GNOME daemon.

### 2.3 Set the color scheme signal

GTK4 and libadwaita apps check this signal to decide light vs dark mode:

```bash
gsettings set org.gnome.desktop.interface color-scheme prefer-dark
gsettings set org.gnome.desktop.interface gtk-theme 'adw-gtk3-dark'
```

This is already in your `hyprland.conf` `exec-once` block so it runs on
every Hyprland start. Run it manually now to apply immediately.

### 2.4 Verify GTK theming

Open a GTK3 app (e.g. `nautilus` or any file manager) and a GTK4 app and
confirm they look consistent. Both should show the dark adw-gtk3 theme.

## 3. Qt theming

Qt uses a plugin system — you configure which style plugin to use, then
point it at a Kvantum theme. GTK themes have no effect here.

### 3.1 Configure qt6ct

```bash
qt6ct
```

In the Appearance tab:
- Style: `kvantum`
- Color scheme: leave as default (Kvantum controls this)

In the Fonts tab set your preferred font.

Click OK.

### 3.2 Configure qt5ct

```bash
qt5ct
```

Same settings as qt6ct — Style: `kvantum`.

### 3.3 Configure Kvantum

```bash
kvantummanager
```

In the main window:
- **Install / Update a Theme** — install a Kvantum theme of your choice.
  KvArc, Lightly, and KvGnomeDark are popular starting points.
- **Change / Delete a Theme** — select your installed theme and click
  **Use this theme**.

Click Close.

### 3.4 Verify Qt theming

Open a Qt app (`qt6ct` itself counts) and confirm it reflects the Kvantum
theme. If it still shows unstyled, check that these env vars are set in
your shell:

```bash
echo $QT_QPA_PLATFORMTHEME   # should output: qt6ct
```

> `QT_STYLE_OVERRIDE` is not used — the style is configured inside qt6ct
> itself. Setting it as an env var causes a warning from qt6ct.

If `QT_QPA_PLATFORMTHEME` is missing, reload your shell:

```bash
source ~/.zshrc
```

## 4. matugen — color pipeline

matugen generates a Material You color palette from your wallpaper and
renders it into template files across all apps. The generated files live
in `~/.cache/matugen/` and are never committed to git.

### 4.1 Configure matugen templates

Edit your stowed matugen config to define what gets generated:

```bash
nvim ~/dotfiles/matugen/.config/matugen/config.toml
```

```toml
# ~/.config/matugen/config.toml

[config]
reload_gtk_theme = true      # trigger GTK reload after generation

# ── Templates ────────────────────────────────────────────────────────────────
# Each template reads your wallpaper colors and writes a generated file.
# The generated files are @imported by your Stow-managed config files.

[templates.waybar-colors]
input_path  = "~/.config/matugen/templates/waybar-colors.css"
output_path = "~/.cache/matugen/colors-waybar.css"

[templates.gtk-colors]
input_path  = "~/.config/matugen/templates/gtk-colors.css"
output_path = "~/.cache/matugen/colors-gtk.css"

[templates.kitty-colors]
input_path  = "~/.config/matugen/templates/kitty-colors.conf"
output_path = "~/.cache/matugen/colors-kitty.conf"

[templates.hyprland-colors]
input_path  = "~/.config/matugen/templates/hyprland-colors.conf"
output_path = "~/.cache/matugen/colors-hyprland.conf"
```

### 4.2 Create the templates directory

```bash
mkdir -p ~/dotfiles/matugen/.config/matugen/templates
```

### 4.3 Write the templates

matugen uses `{{colors.<name>.default.hex}}` syntax to insert color values.
The full color list is available at the matugen docs — the most useful are:

| Variable | Role |
| --- | --- |
| `colors.primary.default.hex` | Main accent color |
| `colors.on_primary.default.hex` | Text on primary |
| `colors.background.default.hex` | App background |
| `colors.on_background.default.hex` | Text on background |
| `colors.surface.default.hex` | Surface / card color |
| `colors.on_surface.default.hex` | Text on surface |
| `colors.secondary.default.hex` | Secondary accent |
| `colors.error.default.hex` | Error / warning color |

**Waybar colors template:**

```bash
nvim ~/dotfiles/matugen/.config/matugen/templates/waybar-colors.css
```

```css
/* Generated by matugen — do not edit directly */
/* Source: ~/dotfiles/matugen/.config/matugen/templates/waybar-colors.css */

@define-color background  {{colors.background.default.hex}};
@define-color surface     {{colors.surface.default.hex}};
@define-color primary     {{colors.primary.default.hex}};
@define-color on_primary  {{colors.on_primary.default.hex}};
@define-color secondary   {{colors.secondary.default.hex}};
@define-color on_surface  {{colors.on_surface.default.hex}};
@define-color error       {{colors.error.default.hex}};
```

**GTK colors template:**

```bash
nvim ~/dotfiles/matugen/.config/matugen/templates/gtk-colors.css
```

```css
/* Generated by matugen — do not edit directly */

@define-color accent_color          {{colors.primary.default.hex}};
@define-color accent_bg_color       {{colors.primary.default.hex}};
@define-color accent_fg_color       {{colors.on_primary.default.hex}};
@define-color window_bg_color       {{colors.background.default.hex}};
@define-color window_fg_color       {{colors.on_background.default.hex}};
@define-color view_bg_color         {{colors.surface.default.hex}};
@define-color view_fg_color         {{colors.on_surface.default.hex}};
@define-color headerbar_bg_color    {{colors.surface.default.hex}};
@define-color headerbar_fg_color    {{colors.on_surface.default.hex}};
@define-color card_bg_color         {{colors.surface_variant.default.hex}};
@define-color card_fg_color         {{colors.on_surface_variant.default.hex}};
```

**Kitty colors template:**

```bash
nvim ~/dotfiles/matugen/.config/matugen/templates/kitty-colors.conf
```

```ini
# Generated by matugen — do not edit directly

background  {{colors.background.default.hex}}
foreground  {{colors.on_background.default.hex}}
cursor      {{colors.primary.default.hex}}

# Black
color0  {{colors.background.default.hex}}
color8  {{colors.surface_variant.default.hex}}

# Red
color1  {{colors.error.default.hex}}
color9  {{colors.error_container.default.hex}}

# Green
color2   {{colors.tertiary.default.hex}}
color10  {{colors.tertiary_container.default.hex}}

# Yellow
color3   {{colors.secondary.default.hex}}
color11  {{colors.secondary_container.default.hex}}

# Blue
color4   {{colors.primary.default.hex}}
color12  {{colors.primary_container.default.hex}}

# Magenta
color5   {{colors.tertiary.default.hex}}
color13  {{colors.on_tertiary_container.default.hex}}

# Cyan
color6   {{colors.on_tertiary.default.hex}}
color14  {{colors.tertiary_container.default.hex}}

# White
color7   {{colors.on_surface.default.hex}}
color15  {{colors.on_background.default.hex}}
```

**Hyprland border colors template:**

```bash
nvim ~/dotfiles/matugen/.config/matugen/templates/hyprland-colors.conf
```

```ini
# Generated by matugen — do not edit directly
# source this from hyprland.conf: source = ~/.cache/matugen/colors-hyprland.conf

$color_active_border   = rgb({{colors.primary.default.hex | replace("#", "")}})
$color_inactive_border = rgb({{colors.surface_variant.default.hex | replace("#", "")}})
```

### 4.4 Import generated colors into config files

**waybar/style.css** — add the import at the top:

```bash
nvim ~/dotfiles/waybar/.config/waybar/style.css
```

```css
@import "/home/<your-username>/.cache/matugen/colors-waybar.css";

/* All structural rules below — no color values hardcoded */
* {
    font-family: "JetBrainsMono Nerd Font";
    font-size: 13px;
}
```

> Replace `<your-username>` with your actual username, or use the full
> path. `~` does not expand inside CSS `@import`.

**gtk-3.0/gtk.css** — add the import:

```bash
nvim ~/dotfiles/gtk/.config/gtk-3.0/gtk.css
```

```css
@import "/home/<your-username>/.cache/matugen/colors-gtk.css";
```

**kitty.conf** — add the import at the bottom:

```bash
nvim ~/dotfiles/kitty/.config/kitty/kitty.conf
```

```ini
# Colors — generated by matugen, imported at runtime
include /home/<your-username>/.cache/matugen/colors-kitty.conf
```

**hyprland.conf** — add the source near the top (after nvidia.conf):

```bash
nvim ~/dotfiles/hypr/.config/hypr/hyprland.conf
```

```ini
source = ~/.config/hypr/nvidia.conf
source = ~/.cache/matugen/colors-hyprland.conf   # add this line
source = ~/.config/hypr/monitors.conf
```

Then use the variables for window borders in hyprland.conf:

```ini
general {
    col.active_border   = $color_active_border $color_secondary 45deg
    col.inactive_border = $color_inactive_border
}
```

### 4.5 Run matugen for the first time

```bash
matugen image ~/pictures/wallpapers/wallpaper.jpg
```

matugen generates the palette, renders all templates, and writes the output
files to `~/.cache/matugen/`. With `reload_gtk_theme = true` in the config
it also signals GTK apps to reload.

Verify the output files were created:

```bash
ls ~/.cache/matugen/
# Should show: colors-waybar.css  colors-gtk.css  colors-kitty.conf  colors-hyprland.conf
```

### 4.6 Reload running apps

```bash
# Reload waybar
pkill waybar && waybar &

# Reload Hyprland config (picks up new border colors)
hyprctl reload

# Kitty reloads automatically when kitty.conf changes
# Open a new kitty window to see the new colors
```

### 4.7 Verify color consistency

Open kitty, waybar, a GTK app, and a Qt app side by side. All should share
the same palette derived from your wallpaper. If any app is still using old
colors, it may need to be restarted.

## 5. Automate matugen on wallpaper change

Rather than running matugen manually every time, hook it into swww so
changing the wallpaper automatically regenerates the palette and reloads
everything.

Create a wallpaper change script in your dotfiles:

```bash
mkdir -p ~/dotfiles/shell/scripts
nvim ~/dotfiles/shell/scripts/wallpaper.sh
```

```bash
#!/usr/bin/env bash
# wallpaper.sh — set wallpaper and regenerate matugen colors
# Usage: wallpaper.sh ~/pictures/wallpapers/my-wallpaper.jpg

WALLPAPER="$1"

if [[ -z "$WALLPAPER" ]]; then
    echo "Usage: wallpaper.sh <path-to-image>"
    exit 1
fi

# Set wallpaper with transition
swww img "$WALLPAPER" \
    --transition-type wipe \
    --transition-duration 1

# Regenerate colors
matugen image "$WALLPAPER"

# Reload apps
pkill waybar && waybar &
hyprctl reload
```

Make it executable:

```bash
chmod +x ~/dotfiles/shell/scripts/wallpaper.sh
```

The `~/dotfiles/shell/scripts` directory is already on your `$PATH` via
the `.zshrc` configured in guide 3. Usage from anywhere:

```bash
wallpaper.sh ~/pictures/wallpapers/wallpaper.jpg
```

## 6. Commit the template files

The templates are part of your dotfiles and should be committed. The
generated output in `~/.cache/matugen/` is not — it's covered by
`.gitignore`.

```bash
cd ~/dotfiles
git add matugen/
git add waybar/ gtk/ kitty/ hypr/
git commit -m "theme: add matugen templates and color imports"
git push
```

## 7. greetd / ReGreet theming

ReGreet is a GTK4 application so it can be themed with a CSS file. This
is the last step — do it only after the rest of the session looks how you
want it, since you need to log out to see the greeter.

### 7.1 Install greetd, cage, and regreet

```bash
paru -S greetd regreet
sudo pacman -S cage
```

### 7.2 Configure greetd

```bash
sudo nvim /etc/greetd/config.toml
```

```toml
[terminal]
vt = 1

[default_session]
# cage -s = primary output only — secondary portrait monitor stays off
command = "cage -s -- regreet"
user = "greeter"
```

### 7.3 Configure ReGreet

```bash
sudo nvim /etc/regreet.toml
```

```toml
[background]
path = "/home/<your-username>/pictures/wallpapers/wallpaper.jpg"
fit = "Cover"

[env]
GTK_THEME = "adw-gtk3-dark"
```

### 7.4 Theme ReGreet with CSS

Create the CSS file:

```bash
sudo nvim /etc/regreet.css
```

```css
/* ReGreet — GTK4 CSS theme */
/* Colors match your session theme via adw-gtk3-dark */

window {
    background-color: alpha(black, 0.4);
}

box.login-box {
    background-color: alpha(white, 0.05);
    border-radius: 12px;
    padding: 32px;
    border: 1px solid alpha(white, 0.1);
}

entry {
    background-color: alpha(white, 0.08);
    border-radius: 8px;
    color: white;
}

button {
    border-radius: 8px;
}
```

> **Note:** ReGreet runs before your session starts so it cannot read from
> `~/.cache/matugen/`. The CSS above uses semi-transparent overlays that
> look good with any wallpaper rather than hardcoded palette colors.
> Once the session is running, your full matugen colors take over.

### 7.5 Test ReGreet before enabling

Test it from a TTY without enabling the service — if it fails you can
still log in normally:

```bash
# Switch to a free TTY
# Then run cage directly
cage -s -- regreet
```

Verify the login box appears on your primary monitor only, the wallpaper
shows, and you can log in successfully.

### 7.6 Enable greetd

Only enable once the test above passes:

```bash
sudo systemctl enable greetd
```

Reboot:

```bash
sudo reboot
```

You should now see ReGreet on your primary monitor. The secondary portrait
monitor stays dark until Hyprland starts and your `monitors.conf` kicks in.

## Stow note — matugen templates

The template files are inside your dotfiles repo and stowed normally. The
generated output is not stowed — it writes directly to `~/.cache/matugen/`
which is outside the stow tree. This is intentional:

```
~/dotfiles/matugen/.config/matugen/
├── config.toml         ← stowed, in git
└── templates/
    ├── waybar-colors.css       ← stowed, in git
    ├── gtk-colors.css          ← stowed, in git
    ├── kitty-colors.conf       ← stowed, in git
    └── hyprland-colors.conf    ← stowed, in git

~/.cache/matugen/               ← generated, NOT in git
├── colors-waybar.css
├── colors-gtk.css
├── colors-kitty.conf
└── colors-hyprland.conf
```
