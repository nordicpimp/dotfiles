# Base Arch install — before anything else

## Preparation

- Get an ISO: [Downloads](https://archlinux.org/download/) — [Acquire an installation image](https://wiki.archlinux.org/title/Installation_guide#Acquire_an_installation_image)
- Get a GitHub PAT: [Tokens (Classic)](https://github.com/settings/tokens)
  - Click `Generate new token` → `Generate new token (classic)`

These steps are assumed complete from here on. PAT stored in an accessible location. ISO flashed to a bootable USB.

---

## 1. Boot from USB and prepare the live environment

### 1.1 Set keyboard layout

```bash
localectl list-keymaps          # list available keymaps
loadkeys se-latin1              # load Swedish keymap
```

### 1.2 Connect to WiFi (live session only)

> **Note:** This connection is only for the live install session. The installed
> system will use NetworkManager — configured separately in step 1.3.
> Do **not** use "Copy network config from live install" in archinstall —
> that would install iwd instead of NetworkManager, which is a different
> incompatible network backend.

```bash
iwctl
station wlan0 connect <SSID>    # enter WiFi password when prompted
exit                            # back to main TTY
```

Verify connectivity:

```bash
ping -c 3 archlinux.org
```

## 2. Run archinstall

```bash
archinstall
```

### 2.1 Archinstall configuration choices

Work through each prompt carefully. The choices below are specific to this hardware
and setup — do not accept defaults without checking.

| Prompt | Choice | Reason |
| --- | --- | --- |
| Archinstall language | English | — |
| Mirrors / region | Sweden (or nearest) | Faster downloads |
| Locale | `sv_SE.UTF-8` or `en_US.UTF-8` | Your preference |
| Disk configuration | See partition layout below | Manual partitioning |
| Bootloader | `systemd-boot` | Clean, UEFI-native, easier to edit than GRUB |
| Unified kernel images | No | Keep standard initramfs — easier to troubleshoot |
| Swap | Yes, size = 32 GiB | Matches installed RAM — required for hibernation |
| Hostname | Your choice | e.g. `archbox` or `ws01-arch-local` |
| Root password | Set a strong password | — |
| User account | Create your user, add to `wheel` group | — |
| Profile | **Minimal** | Do NOT select GNOME, KDE, or any DE — Hyprland is installed manually after |
| Audio | `pipewire` | Required for the PipeWire + WirePlumber stack |
| Network | **NetworkManager** | See note above — do NOT choose "Copy network config from live install" |
| Timezone | Europe/Stockholm | Or your timezone |
| NTP | Yes | — |
| Additional packages | `base-devel git` | Needed for AUR helper and building packages |

### 2.2 Partition layout (UEFI/GPT)

Inspired by the [Arch wiki partition layout](https://wiki.archlinux.org/title/Partitioning#Example_layouts).

| Mount point | Partition | Type GUID | Suggested size |
| --- | --- | --- | --- |
| `/boot` | `/dev/sda1` | `C12A7328-F81F-11D2-BA4B-00A0C93EC93B` EFI system | 1 GiB |
| `[SWAP]` | `/dev/sda2` | `0657FD6D-A4AB-43C4-84E5-0933C84B4F4F` Linux swap | 32 GiB (= RAM, needed for hibernation) |
| `/` | `/dev/sda3` | `4F68BCE3-E8CD-4DB1-96E7-FBCAF984B709` Linux x86-64 root | 60–100 GiB |
| `/home` | `/dev/sda4` | `933AC7E1-2EB4-4F13-B844-0E14E2AEF915` Linux /home | Remainder of disk |

> `/dev/sda` is an example. Your actual disk identifier may be `/dev/nvme0n1`
> or similar — verify with `lsblk` before partitioning.

### 2.3 Confirm and install

Review the summary archinstall shows before confirming. Verify:

- Profile is **Minimal** (no desktop environment)
- Network shows **NetworkManager**
- Bootloader is **systemd-boot**
- `base-devel` and `git` are in the package list

Confirm and let archinstall run. When it finishes:

> **Do NOT reboot yet.** Proceed to step 3 to configure NVIDIA before the
> first boot. Rebooting now without NVIDIA configured may result in a broken
> session that is difficult to recover from.

## 3. Post-install: NVIDIA configuration (before first reboot)

Stay in the live environment. Chroot into the newly installed system:

```bash
arch-chroot /mnt
```

You are now operating inside your new install. All commands below affect the
real system, not the live ISO.

### 3.1 Enable multilib repository

`lib32-nvidia-utils` is a 32-bit package that lives in the `multilib` repo,
which is disabled by default. Without enabling it, pacman silently skips it —
Steam and Proton will fail later with confusing errors.

Edit `/etc/pacman.conf`:

```bash
nvim /etc/pacman.conf
```

Find these two lines (they are commented out) and remove the `#` from both:

```
# Before:
#[multilib]
#Include = /etc/pacman.d/mirrorlist

# After:
[multilib]
Include = /etc/pacman.d/mirrorlist
```

Save and sync package databases:

```bash
pacman -Sy
```

### 3.2 Install NVIDIA driver stack

The RTX 4070 Ti Super is Ada Lovelace architecture — use the open kernel modules.

```bash
pacman -S nvidia-open nvidia-utils lib32-nvidia-utils nvidia-settings
```

### 3.3 Configure initramfs

Edit `/etc/mkinitcpio.conf`:

```bash
nvim /etc/mkinitcpio.conf
```

Find the `MODULES=()` line and update it:

```
MODULES=(nvidia nvidia_modeset nvidia_uvm nvidia_drm)
```

Also find the `HOOKS=` line and remove `kms` from it if present — having
`kms` alongside the NVIDIA modules causes conflicts.

Rebuild the initramfs:

```bash
mkinitcpio -P
```

### 3.4 Add kernel parameters to bootloader

With systemd-boot, the entry file is at `/boot/loader/entries/`.
Find the Arch entry:

```bash
ls /boot/loader/entries/
```

Edit the file (name will vary, typically `arch.conf` or similar):

```bash
nvim /boot/loader/entries/arch.conf
```

Find the `options` line and append the NVIDIA parameters:

```
# Example — your options line may already have root= and rw, append after those:
options root=UUID=xxxx rw nvidia_drm.modeset=1 nvidia_drm.fbdev=1 nvidia.NVreg_PreserveVideoMemoryAllocations=1
```

> `nvidia.NVreg_PreserveVideoMemoryAllocations=1` prevents graphical
> corruption when resuming from suspend or hibernate.

### 3.5 Enable NVIDIA power management services

```bash
systemctl enable nvidia-suspend
systemctl enable nvidia-hibernate
systemctl enable nvidia-resume
```

### 3.6 Exit chroot and reboot

```bash
exit            # leave the chroot
reboot          # first boot into the real system
```

Remove the USB when the machine restarts.

## 4. First boot verification

Log in as your user. Verify the system is in good shape before proceeding
to Hyprland setup.

```bash
# Verify NVIDIA driver is loaded correctly
nvidia-smi

# Verify NetworkManager is running
systemctl status NetworkManager

# Connect to WiFi (now using NetworkManager, not iwd)
nmtui

# Verify multilib is enabled and 32-bit libs are present
pacman -Ss lib32-nvidia-utils | grep installed

# Check for any failed systemd services
systemctl --failed
```

If `nvidia-smi` shows your GPU and driver version, NVIDIA is configured
correctly. If it errors, review steps 3.2–3.4 before continuing.

## 5. Install tmux — split screen and copy-paste in the TTY

The TTY has no native clipboard and no split screen. tmux solves both — it
runs inside a TTY and gives you split panes, copy mode, and multiple windows.
Install it now so the rest of the setup is less painful to follow along with.

```bash
sudo pacman -S tmux
```

Start a session:

```bash
tmux
```

Recommended layout for following these guides — frogmouth in one pane,
commands in the other:

```
Ctrl+B %          split into left and right panes
Ctrl+B ←→         move between panes
```

In the left pane open your guide, in the right pane run commands. Text copied
in copy mode can be pasted into either pane.

| Keys | Action | Swedish keyboard |
| --- | --- | --- |
| `Ctrl+B %` | Split vertically (left / right) | same |
| `Ctrl+B "` | Split horizontally (top / bottom) | same |
| `Ctrl+B ←→↑↓` | Move between panes | same |
| `Ctrl+B z` | Zoom pane to full screen (toggle) | same |
| `Ctrl+B [` | Enter copy mode | `Ctrl+B` then `AltGr+8` |
| `Space` | Start selection in copy mode | same |
| `Enter` | Copy selection | same |
| `Ctrl+B ]` | Paste | `Ctrl+B` then `AltGr+9` |
| `Ctrl+B d` | Detach session (keeps running in background) | same |
| `tmux attach` | Reattach to detached session | same |

> tmux sessions survive if you accidentally close the terminal or switch TTYs.
> Run `tmux attach` to get back to exactly where you were.

## 6. Install AUR helper

The AUR helper cannot be installed as root and is not in the official repos —
it must always be built manually as your user after first boot.

**Why paru over yay:** paru never caches sudo credentials between operations,
so you always know exactly when elevated privileges are being used. It also
opens the PKGBUILD for review before building, and shows Arch news on updates.

```bash
# Install paru (AUR helper)
git clone https://aur.archlinux.org/paru.git
cd paru
makepkg -si
cd ..
rm -rf paru
```

All subsequent AUR packages are installed with `paru -S <package>`.

## 7. Install frogmouth — markdown reader for the TTY

All remaining guides are markdown files. Install frogmouth now so you can read
them from the terminal without needing a browser or desktop environment.

```bash
paru -S frogmouth
```

Mount your USB if it is not already mounted (udiskie is not installed yet at
this stage — mount manually):

```bash
# Find your USB device name
lsblk

# Mount it (replace sdX1 with your actual device)
sudo mkdir -p /mnt/usb
sudo mount /dev/sdX1 /mnt/usb
```

Open a guide:

```bash
frogmouth /mnt/usb/Configure/2.hyprland-install.md
```

Frogmouth keybinds:

| Key | Action |
| --- | --- |
| `q` | Quit |
| `b` | Back |
| Arrow keys / `hjkl` | Navigate |
| `Tab` | Switch between file browser and content |
| `Enter` | Open file or follow link |
| `Ctrl+L` | Open a file path directly |

> From this point on, open each guide in frogmouth before starting it.
> You can keep frogmouth open in one kitty split and work in another once
> Hyprland is running.
