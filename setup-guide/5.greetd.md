# greetd — configuration, troubleshooting, and recovery

This guide covers how greetd works under the hood, the greeter user, session
file registration, common issues, and how to recover if the login manager
breaks.

**Prerequisites:** `4.theming.md` complete. greetd installed and enabled.
ReGreet appearing on your primary monitor at boot.

---

## 1. How greetd works

Understanding the architecture makes troubleshooting much easier.

```
systemd
  └── greetd.service  (runs as root, manages the display)
        └── cage -s   (micro-compositor, opens on primary monitor only)
              └── regreet  (GTK4 UI — the actual login screen you see)
                    └── on successful login:
                          └── uwsm / start-hyprland  (your session)
```

**greetd** is a minimal daemon that manages the login process. It runs as
root via PAM (Pluggable Authentication Modules) and is responsible for
authenticating users and starting their sessions. It has no UI of its own.

**cage** is a single-application Wayland compositor. It starts a minimal
Wayland environment just for the greeter. The `-s` flag restricts it to the
primary output — your portrait monitor gets no signal until Hyprland starts.

**regreet** is a GTK4 application that provides the login form. It runs
inside cage, authenticates via greetd's IPC, and launches the selected
session on success.

The key difference from a regular session: greetd and cage run as a
dedicated `greeter` system user, not as you. This is a security boundary —
the login screen cannot access your files.

## 2. The greeter user

greetd creates a `greeter` system user during installation. You can verify:

```bash
id greeter
# Should show: uid=... gid=... groups=...,video
```

The `greeter` user needs to be in the `video` group to access the display.
If it is missing:

```bash
sudo usermod -aG video greeter
```

The greeter user has no home directory by default. Config files for greetd
and regreet live in `/etc/` rather than `~/.config/` for this reason.

## 3. Session files — making Hyprland appear in the session list

ReGreet populates its session selector from `.desktop` files in
`/usr/share/wayland-sessions/`. Hyprland installs one automatically:

```bash
cat /usr/share/wayland-sessions/hyprland.desktop
```

Expected output:

```ini
[Desktop Entry]
Name=Hyprland
Comment=An intelligent dynamic tiling Wayland compositor
Exec=Hyprland
Type=Application
```

If this file is missing or `Exec` points to the wrong binary, ReGreet will
not show Hyprland in the session list. You can create or fix it manually:

```bash
sudo nvim /usr/share/wayland-sessions/hyprland.desktop
```

```ini
[Desktop Entry]
Name=Hyprland
Comment=An intelligent dynamic tiling Wayland compositor
Exec=start-hyprland
Type=Application
```

> **Exec=start-hyprland** rather than bare `Hyprland` — consistent with
> how we launch from the TTY, using uwsm for proper session management.

After saving, the session should appear in ReGreet's dropdown on next boot
or after restarting greetd.

## 4. Config file reference

All greetd and ReGreet configuration lives in `/etc/`. None of it is in
your dotfiles since these files require root ownership.

| File | Purpose |
| --- | --- |
| `/etc/greetd/config.toml` | greetd daemon config — which compositor and greeter to launch |
| `/etc/regreet.toml` | ReGreet config — wallpaper, environment, GTK theme |
| `/etc/regreet.css` | ReGreet CSS theme |
| `/usr/share/wayland-sessions/*.desktop` | Available sessions shown in ReGreet dropdown |

### Full /etc/greetd/config.toml reference

```toml
[terminal]
vt = 1          # virtual terminal to use — 1 is standard

[default_session]
command = "cage -s -- regreet"
user = "greeter"
```

### Full /etc/regreet.toml reference

```toml
[background]
path = "/home/<your-username>/pictures/wallpapers/wallpaper.jpg"
fit = "Cover"       # Cover | Contain | Fill | ScaleDown | None

[GTK]
application_prefer_dark_theme = true
icon_theme = "Papirus-Dark"
font_name = "Sans 11"
cursor_theme = "Bibata-Modern-Ice"

[env]
GTK_THEME = "adw-gtk3-dark"
```

## 5. Updating the ReGreet wallpaper

When you change your main wallpaper via the `wallpaper.sh` script, the
ReGreet background does not update automatically — it reads a static path
from `/etc/regreet.toml`.

Options:

**Option A — point regreet.toml at a fixed path and symlink your wallpaper there:**

```bash
sudo nvim /etc/regreet.toml
# Set path = "/etc/greetd/wallpaper.jpg"

# Then update this file when you change wallpaper
sudo cp ~/pictures/wallpapers/new-wallpaper.jpg /etc/greetd/wallpaper.jpg
```

**Option B — update regreet.toml as part of your wallpaper script:**

Add to `~/dotfiles/shell/scripts/wallpaper.sh`:

```bash
# Update greetd wallpaper (requires sudo)
sudo sed -i "s|path = \".*\"|path = \"$WALLPAPER\"|" /etc/regreet.toml
```

## 6. Restarting greetd without rebooting

If you change `/etc/greetd/config.toml` or `/etc/regreet.toml` and want
to apply without rebooting:

```bash
sudo systemctl restart greetd
```

> **Warning:** restarting greetd from inside a running session will kill
> your session. Run this only from a TTY where you are not logged into
> Hyprland, or accept that your session will end.

To apply only CSS changes without restarting, log out and back in — ReGreet
re-reads the CSS file each time it launches.

## 7. Troubleshooting

### ReGreet does not appear — black screen on boot

Check the greetd journal:

```bash
sudo journalctl -u greetd -n 50
```

Common causes:

| Symptom in journal | Fix |
| --- | --- |
| `cage: failed to open display` | greeter user not in video group — `sudo usermod -aG video greeter` |
| `regreet: failed to connect to greetd` | greetd socket permission issue — check `/run/greetd.sock` |
| `cage: no outputs available` | NVIDIA DRM not initialised — verify kernel params from guide 1 |
| `error: /etc/regreet.toml not found` | regreet not installed or config missing |

### Login succeeds but Hyprland fails to start

greetd launched regreet correctly and authenticated you, but the session
command failed. Check:

```bash
sudo journalctl -u greetd -n 100 | grep -i "error\|fail\|exit"
```

Also check whether the session file `Exec` command works from the greeter
user's context:

```bash
# Test as greeter user
sudo -u greeter /bin/bash -c "which start-hyprland"
```

If `start-hyprland` is not found, check that it is in a standard PATH
location (`/usr/bin/` or `/usr/local/bin/`).

### Wrong monitor — greeter appears on portrait display

The `-s` flag on cage selects the "focused" output, which is determined
by the GPU's connector order. If cage is picking the wrong monitor:

```bash
# Check which connectors the GPU reports and in what order
ls /sys/class/drm/ | grep card
```

You can force a specific output by replacing `-s` with an explicit output
name in `/etc/greetd/config.toml`:

```toml
command = "cage -s -m DP-1 -- regreet"
```

Replace `DP-1` with your actual primary monitor name from `hyprctl monitors`.

### Session dropdown is empty in ReGreet

No `.desktop` files found in `/usr/share/wayland-sessions/`:

```bash
ls /usr/share/wayland-sessions/
```

If empty or missing `hyprland.desktop`, create it as shown in section 3.

### greetd enabled but autologin was also configured

If you previously set up TTY autologin (from before greetd) and greetd is
now enabled, they may conflict. Check:

```bash
sudo systemctl status getty@tty1
```

If it shows a conflicting autologin override, disable it:

```bash
sudo systemctl edit getty@tty1
# Remove any autologin override lines, save and exit
sudo systemctl daemon-reload
```

## 8. Recovery — if greetd breaks and you cannot log in

If greetd fails and you are presented with a black screen or error, you can
always recover via a different TTY.

**Method 1 — switch to another TTY:**

At any point, even if greetd is showing an error:

```
Ctrl + Alt + F3
```

Log in as your user on TTY3. From here you can fix the config:

```bash
sudo nvim /etc/greetd/config.toml
sudo systemctl restart greetd
```

**Method 2 — disable greetd temporarily and use TTY login:**

```bash
sudo systemctl disable greetd
sudo reboot
# Normal TTY login will appear
# Fix the issue, then re-enable:
sudo systemctl enable greetd
```

**Method 3 — boot from Arch ISO and chroot:**

If you cannot log in at all (e.g. forgot password, broken PAM config):

```bash
# Boot from Arch ISO, then:
arch-chroot /mnt
passwd <your-username>    # reset password
# or fix whatever config is broken
exit
reboot
```

## 9. Disabling greetd (reverting to TTY login)

If you ever want to go back to plain TTY login:

```bash
sudo systemctl disable greetd
```

Add this to `~/.zprofile` to auto-launch Hyprland on TTY1 login:

```bash
if [ -z "$DISPLAY" ] && [ "$XDG_VTNR" -eq 1 ]; then
    exec start-hyprland
fi
```

This is the no-display-manager approach — fastest boot, no overhead, but
a plain terminal prompt instead of a graphical login screen.

## 10. Maintenance

**Updating ReGreet:**

```bash
paru -Su regreet
```

ReGreet is an AUR package and updates independently from the system.
Check for breaking changes in the ReGreet changelog before updating —
the config format occasionally changes between versions.

**Checking greetd version:**

```bash
greetd --version
```

**Viewing full greetd logs:**

```bash
sudo journalctl -u greetd --since today
```
