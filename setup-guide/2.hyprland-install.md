# Hyprland install — core session setup

This guide installs Hyprland and the minimum packages needed to get a working
session. The goal at the end of this file is a confirmed, stable Hyprland
session launched manually from a TTY — nothing more. Dotfiles, theming, and
greetd come in later guides.

**Prerequisites:** `1.base-arch-install.md` complete. System booted, NVIDIA
verified, paru installed, NetworkManager running.

---

## 1. Install core session packages

### 1.1 Official repo packages

```bash
sudo pacman -S \
  hyprland \
  uwsm \
  xdg-desktop-portal-hyprland \
  xdg-desktop-portal \
  xorg-xwayland \
  qt5-wayland \
  qt6-wayland \
  pipewire-pulse \
  pipewire-alsa \
  pipewire-jack \
  kitty \
  polkit

> **Why uwsm:** Universal Wayland Session Manager. `start-hyprland` is a
> wrapper around it that launches Hyprland as a proper systemd session.
> Running bare `Hyprland` still works but produces a warning telling you to
> use this method instead. uwsm also handles session cleanup correctly on exit.
```

> **Why xorg-xwayland:** Steam, some games, and other X11-only apps run through
> this compatibility layer. Required even on a Wayland-native setup.

> **Why polkit:** Handles privilege escalation dialogs (e.g. mounting drives,
> package manager GUIs). Without it many apps silently fail to do things that
> require elevated permissions. `hyprpolkitagent` (installed later) depends on
> this being present.

### 1.2 AUR packages

```bash
paru -S \
  hyprland-qt-support \
  hyprqt6engine \
  hyprpolkitagent \
  hypridle \
  hyprlock \
  hyprpicker \
  hyprcursor
```

## 2. Enable PipeWire

PipeWire must be started as a user service, not a system service.

```bash
systemctl --user enable --now pipewire pipewire-pulse wireplumber
```

Verify all three are running:

```bash
systemctl --user status pipewire pipewire-pulse wireplumber
```

All three should show `active (running)`. If any fail, check the journal:

```bash
journalctl --user -u pipewire -n 50
```

## 3. Create a minimal Hyprland config

This is a temporary config to get into a working session. It will be replaced
by your stowed dotfiles config in `3.dotfiles.md`. Do not invest time in
configuring this file — it only needs to be functional enough to verify the
session works.

```bash
mkdir -p ~/.config/hypr
nvim ~/.config/hypr/hyprland.conf
```

Paste the following minimal config:

```ini
# Minimal config — temporary, replaced by dotfiles in guide 3

# NVIDIA required env vars
env = LIBVA_DRIVER_NAME,nvidia
env = __GLX_VENDOR_LIBRARY_NAME,nvidia
env = GBM_BACKEND,nvidia-drm
env = NVD_BACKEND,direct
env = ELECTRON_OZONE_PLATFORM_HINT,auto

# Monitors — adjust resolution and refresh rate to match your displays
# Format: monitor=name,resolution@hz,position,scale
monitor=,preferred,auto,1

# Input
input {
    kb_layout = se
    follow_mouse = 1
    touchpad {
        natural_scroll = false
    }
}

# Basic keybinds — enough to open a terminal and exit
$mod = SUPER
bind = $mod, Return, exec, kitty
bind = $mod SHIFT, Q, killactive
bind = $mod SHIFT, E, exit

# Launch polkit agent and automounter on start
exec-once = /usr/lib/hyprland/hyprpolkitagent
exec-once = udiskie
```

> **Monitor name:** to find your actual monitor identifier, you can check
> `ls /sys/class/drm/` after booting, or use the fallback `monitor=,preferred,auto,1`
> which auto-detects. Your real monitor config goes in your dotfiles later.

## 4. Launch Hyprland from TTY

Log out of any current session if needed. Switch to a TTY:

```
Ctrl + Alt + F2
```

Log in as your user. Start Hyprland:

```bash
start-hyprland
```

> `start-hyprland` is a wrapper script included with the hyprland package.
> It uses `uwsm` to launch Hyprland as a proper systemd session.
> Do not use bare `Hyprland` — it still works but warns you to use this instead.

---

## 5. Verify the session

Once inside Hyprland, press `Super + Return` to open kitty. Run these checks:

```bash
# Confirm Wayland session is active
echo $WAYLAND_DISPLAY        # should output: wayland-1 (or similar)
echo $XDG_SESSION_TYPE       # should output: wayland

# Confirm NVIDIA is driving the session
nvidia-smi                   # should show GPU and driver version

# Check Hyprland can see your monitors
hyprctl monitors             # should list both displays

# Confirm PipeWire is running
pactl info | grep Server     # should show PulseAudio server via PipeWire

# Check for any Hyprland errors
hyprctl -j errors
```

> **Dual monitor check:** `hyprctl monitors` should list both your primary
> (landscape) and secondary (portrait) monitors. If the portrait monitor shows
> incorrectly rotated, that is fine for now — monitor layout is configured
> in your dotfiles.

If Hyprland started, kitty opened, and the checks above pass — the session is
confirmed stable. You have a working Hyprland install on NVIDIA.

---

## 6. Check the Hyprland log for warnings

Even if everything looks fine visually, check the log for anything worth
knowing before moving on:

```bash
cat /tmp/hypr/$(ls /tmp/hypr)/hyprland.log | grep -i "error\|warn" | tail -30
```

Common warnings that are safe to ignore at this stage:
- Missing config file warnings for files you haven't created yet
- Missing cursor theme warnings — cursor is configured in dotfiles
- XDG portal warnings — these resolve once the portal is running fully

Warnings about NVIDIA or DRM are worth investigating before proceeding.

---

## 7. Exit and return to TTY

When satisfied, exit Hyprland:

```
Super + Shift + E
```

Or from kitty:

```bash
hyprctl dispatch exit
```
